## Part 1

Given a list of instructions for turning a 100 position circular safe dial (numbered 0 to 99), track the position of the dial as we spin it left and right, and count how many times it lands on exactly 0.

Given a sequence of instructions for turning a 100 position circular dial, calculate the password by totaling every time the dial passes through 0, including full rotations and partial movements that cross the zero mark during the transition, rather than just checking its final landing spot.

## Solution

First, data structures are defined to contain data. One to keep the dial's state and one to keep the state of the process.

```
type Dial struct {
	currentPosition int
	smallestIndex   int
	highestIndex    int
	totalIndices    int
}

type Simulation struct {
	dial             *Dial
	instructions     []int
	startingPosition int
	target           int
	landsOnTarget    int
	clicksOnTarget   int
}

func NewDial(smallest int, highest int) *Dial {
	return &Dial{
		currentPosition: 0,
		smallestIndex:   smallest,
		highestIndex:    highest,
		totalIndices:    highest - smallest + 1,
	}
}

func NewSimulation(d *Dial, instructions []int, startingPosition, target int) *Simulation {
	return &Simulation{
		dial:             d,
		instructions:     instructions,
		startingPosition: startingPosition,
		target:           target,
		landsOnTarget:    0,
		clicksOnTarget:   0,
	}
}
```

With the data structures defined, a function is created to parse the data needed. Takes each line from a text file and extracts direction and distance e.g. R56 becomes +56, and L32 becomes -32.

```
func ParseInstructions(filename string) (instructions []int, err error) {

	file, err := os.Open(filename)
	if err != nil {
		return nil, fmt.Errorf("File opening failure: %w", err)
	}
	defer file.Close()

	scanner := bufio.NewScanner(file)

	for scanner.Scan() {
		line := scanner.Text()

		distance, err := strconv.Atoi(line[1:])
		if err != nil {
			return nil, fmt.Errorf("line %s: %w", line, err)
		}

		direction := line[0]

		switch direction {
		case 'R':
			instructions = append(instructions, distance)
		case 'L':
			instructions = append(instructions, -distance)
		default:
			return nil, fmt.Errorf("invalid direction '%c' on line: %s", direction, line)
		}

	}

	if err := scanner.Err(); err != nil {
		return nil, fmt.Errorf("scan: %w", err)
	}

	return instructions, nil
}
```

The implementation logic is then defined. First the absolute value is taken to find how many initial full turns (`fullTurns := distance / s.dial.totalIndices`) would complete. Next, the remainder (`remainder := distance % s.dial.totalIndices`) is calculated to evaluate whether an additional partial turn reaches the boundary. If it does, one more is added to the count. This count will provide the answer to Part 2.

Another function, `func Wrap()`, is used to calculate the dial's new position. This allows the number of times the dial rests on 0 to be counted in order to solve part 1.

```
func (s *Simulation) Run() {

	s.dial.currentPosition = s.startingPosition

	for _, move := range s.instructions {
		distance := move
		if move < 0 {
			distance = -distance
		}

		fullTurns := distance / s.dial.totalIndices
		remainder := distance % s.dial.totalIndices

		s.clicksOnTarget += fullTurns

		if move > 0 {
			if s.dial.currentPosition+remainder > s.dial.highestIndex {
				s.clicksOnTarget++
			}
		} else {
			if remainder != 0 && s.dial.currentPosition != 0 && s.dial.currentPosition-remainder <= s.dial.smallestIndex {
				s.clicksOnTarget++
			}
		}

		s.dial.currentPosition = s.Wrap(s.dial.currentPosition + move)

		if s.dial.currentPosition == 0 {
			s.landsOnTarget++
		}
	}
}
```

## Notes on Modulus on Negative Numbers

In Go, % doesn't mean modulo, it means remainder. Because of this, it cannot be used directly to find the modulus of negative numbers.

For example, -1 % 5 mathematically should be 4, but Go will evaluate it to be -1 because it truncates towards 0.

To solve this, first evaluate the expression and see whether the result is negative or positive (including the 0 case).

If it is positive, the answer is correct. If it is negative, add the divisor, which wraps the answer back into a positive value.

For example, -1 % 5 evaluates to -1, which is negative, so adding the divisor (-1 + 5) will give the correct answer of 4.

```
func modulo(a, b int) int {
    r := a % b
    if r < 0 {
        return r + b
    }
    return r
}
```

Simplifying further, adding b to the initial remainder allows the result to be wrapped back into bounds when the modulus is applied again. This eliminates the need for branching.

In other words,

- a % b: The initial remainder.

- (a % b) + b: Adding b ensures a negative result becomes positive, and a positive result gets pushed forward beyond the next boundary.

- ((a % b) + b) % b: The final modulo has no effect on the originally negative result, but safely wraps the positive result back into the correct boundary.

```
func Wrap(a, b int) int {
	return ((a % b) + b) % b
}
```

In this code, this becomes

```
func (s *Simulation) Wrap(number int) int {
	return ((number % s.dial.totalIndices) + s.dial.totalIndices) % s.dial.totalIndices
}
```

The `main` function instantiates the `Dial` and injects it into the simulation along with the instructions parsed from the file. The results for part 1 and part 2 are then printed.

```
func main() {

	dial := NewDial(0, 99)

	instructions, err := ParseInstructions("input.txt")
	if err != nil {
		fmt.Println("Error parsing instructions:", err)
	}

	sim := NewSimulation(dial, instructions, 50, 0)

	sim.Run()

	fmt.Printf("Part 1: Dial lands on %d %d times\n", sim.target, sim.landsOnTarget)
	fmt.Printf("Part 2: Dial clicks on %d %d times\n", sim.target, sim.clicksOnTarget)
}
```
